7 Fatal API Mistakes You're Probably Making

A guide to moving beyond "it works on my machine" to production-ready security.

Venom March 17, 2026 • 8 min read

1. Lack of Input Validation

Trusting user-provided data is the fastest way to compromise your system. Validation must happen before any business logic executes.

The Mistake
app.post('/update-profile', (req, res) => {
  db.users.update(req.body); 
  res.send("Done");
});
const profileSchema = zod.object({
  username: zod.string().min(3).max(20),
  age: zod.number().int().positive()
});

app.post('/update-profile', async (req, res) => {
  const data = profileSchema.parse(req.body);
  if (!data.success) return res.status(400).send({message: "Bad request"});
  await db.users.update(req.body);
  res.status(200).json({ success: true });
});

2. Broken Object Level Authorization (BOLA)

Just because a user is authenticated doesn't mean they are authorized to access every ID in your database.

The Mistake
app.get('/api/orders/:id', async (req, res) => {
  const order = await db.orders.findById(req.params.id);
  res.json(order);
});
app.get('/api/orders/:id', async (req, res) => {
  const order = await db.orders.findOne({
    _id: req.params.id,
    userId: req.user.id
  });

  if (!order) return res.status(404).json({ error: "Order not found" });
  res.json(order);
});

3. Missing Rate Limiting

Endpoints without limits are vulnerable to DDoS and brute-force attacks. Use flexible strategies to protect your infrastructure.

import { RateLimiterMemory } from 'rate-limiter-flexible';

const rateLimiter = new RateLimiterMemory({
  points: 10, 
  duration: 1 
});

const rateLimitMiddleware = (req, res, next) => {
  rateLimiter.consume(req.ip)
    .then(() => next())
    .catch(() => res.status(429).send('Too Many Requests'));
};

app.use('/api/', rateLimitMiddleware);

4. Improper Data Filtering

Avoid leaking sensitive internal fields by strictly defining what your API returns.

The Mistake
const user = await db.users.find(id);
res.json(user);
const user = await db.users.find(id);
const { password, internal_id, ssn, ...publicData } = user._doc;

res.json(publicData);

5. Misusing HTTP Methods

Adhering to REST semantics prevents caching issues and unexpected side effects from pre-fetching browsers.

// Correct Semantics
GET    /users     -> Read
POST   /users     -> Create
PATCH  /users/:id -> Partial Update
DELETE /users/:id -> Remove

6. Leaking Stack Traces

Raw error messages provide a map of your infrastructure to potential attackers.

app.use((err, req, res, next) => {
  logger.error(err);
  res.status(500).json({
    message: "Internal Server Error",
    trackingId: req.id
  });
});

7. Lack of API Versioning

Always version your API to ensure backward compatibility as your data structures evolve.

import v1Routes from './routes/v1';
import v2Routes from './routes/v2';

app.use('/api/v1', v1Routes);
app.use('/api/v2', v2Routes);
